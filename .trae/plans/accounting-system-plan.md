# 计划：账套管理系统功能完善计划

## 目标
基于现有前后端分离 + MySQL 的账套管理系统框架，完善功能，使其能够通过导入固定格式的序时账和科目余额表来生成完整的网页版账套，支持科目层级展开/折叠、科目序时账穿透查询、凭证样式展示、本地存储持久化等功能。

## 当前项目状态
项目已完成基础框架搭建：
- **后端**: FastAPI + SQLAlchemy(Async) + aiomysql + MySQL
- **前端**: Vue 3 + Element Plus + Axios + Vite
- **数据库模型**: Book(账套), BalanceSubject(科目余额), JournalEntry(序时账分录)
- **已有功能**: 文件上传解析、科目树展示、余额明细查看、序时账查看、凭证查看、账套管理
- **已有雏形**: 核算维度面板（DimensionPanel.vue，按类型分组展示）
- **待实现/优化的核心功能**:
  - Excel解析器适配实际文件格式（合并单元格表头）
  - 主页面科目余额表（网格表格样式，支持层级折叠）
  - 双击科目穿透到序时账
  - 核算维度独立子界面（从序时账分录提取维度，空白不显示）
  - 凭证样式优化（传统凭证格式）
  - 底稿导出预留功能（拉取科目余额表数据到指定底稿）
  - 本地存储持久化

## Excel 文件格式分析

### 科目余额表 (`科目余额表-20260512100038533-审计账号.xlsx`)
- Sheet: `sheet1`，共 2263 行，11 列
- 表头结构(第3~4行，合并单元格)：
  ```
  科目编码 | 科目名称 | 年初余额      | 本期发生额      | 本年累计      | 期末余额      | 核算维度
                       借方/贷方     | 借方/贷方       | 借方/贷方     | 借方/贷方
  ```
- 科目编码使用 `.` 分隔层级，如 `1123`(一级)、`1123.99`(二级)、`2201.01.01`(三级)
- 支持核算维度字段（银行账户、供应商、客户、项目等）

### 序时账/凭证 (`引出列表_凭证_0512141615.xlsx`)
- Sheet: `列表数据`，共 2882 行，9 列
- 表头: `核算组织 | 期间 | 凭证号 | 摘要 | 科目编码 | 科目名称 | 借方 | 贷方 | 核算维度`
- 同一凭证号的多行分录组成一张完整凭证
- 借方/贷方可能出现负值（表示红字冲销）

## 功能计划

### 阶段一：Excel解析适配（高优先级）

#### 1. 修复科目余额表解析器
- **问题**: 当前解析器搜索 `年初余额（借方）` 等关键词，但实际Excel的合并单元格仅包含 `年初余额`，子栏目标题为 `借方金额`/`贷方金额`
- **解决方案**: 
  - 改进 `find_header_row` 识别两级表头结构
  - 支持识别合并单元格模式
  - 增加对 `借方金额`/`贷方金额` 作为子表头关键词的支持
  - 确保 `col_keywords` 能匹配实际文件中的表头文本
- **涉及文件**: `backend/services/parser.py`

#### 2. 改进序时账解析器
- **当前状态**: 基本适配良好，列名匹配工作正常
- **优化点**:
  - 处理借方/贷方中的负值（红字冲销），保留原始正负号
  - 支持同一凭证号多行分录的组合展示
- **涉及文件**: `backend/services/parser.py`

### 阶段二：科目余额表主页面（高优先级）

#### 3. 主页面展示科目余额表（网格样式）
- **玩法**: 主页面以表格形式显示完整科目余额表
- **列**: 科目编码 | 科目名称 | 期初借方 | 期初贷方 | 本期借方 | 本期贷方 | 期末借方 | 期末贷方
- **数据来源**: 从 `BalanceSubject` 表中读取，按科目编码排序
- **交互**:
  - 双击科目名称行跳转到该科目的序时账
  - 支持科目层级折叠/展开
- **涉及文件**: `frontend/src/components/BalanceDetail.vue`（重构为表格）、`backend/routers/balance.py`（可能需要新增全表接口）

#### 4. 侧栏科目树优化
- **当前状态**: 已有 `SubjectTree.vue` 使用 `el-tree` 展示科目树
- **优化点**:
  - 科目树仅从科目余额表生成，不预设科目
  - 根据科目编码的 `.` 分隔自动确定层级（一级、二级、三级等）
  - 支持自由展开/折叠
  - 点击科目树节点 → 主页面切换到该科目详情
- **涉及文件**: `frontend/src/components/SubjectTree.vue`

### 阶段三：科目序时账穿透查询（高优先级）

#### 5. 双击科目跳转序时账
- **玩法**: 在主科目余额表或科目树中双击科目名称
- **逻辑**: 
  - 如果是一级科目（如 `1002`），显示该一级科目下所有子科目的序时账
  - 如果是二级科目（如 `1123.99`），只显示该二级科目的序时账
  - 如果是三级科目（如 `2201.01.01`），只显示该三级科目的序时账
- **后端接口**: 需要新增或修改接口支持"某一级科目下所有明细科目的序时账合并查询"
- **涉及文件**: `backend/routers/journal.py`、`backend/routers/balance.py`

#### 6. 序时账表格展示
- **当前状态**: 已有 `JournalDetail.vue` 表格展示
- **优化点**:
  - 显示列：期间、凭证号、摘要、科目编码、科目名称、借方金额、贷方金额
  - 点击凭证号弹出凭证详情对话框
  - 支持按期间、凭证号筛选
- **涉及文件**: `frontend/src/components/JournalDetail.vue`

### 阶段四：核算维度独立子界面（高优先级）

#### 7. 核算维度子界面
- **玩法**: 通过一个独立子界面查看指定科目的核算维度
- **当前状态**: 已有 `DimensionPanel.vue` 基础组件，按类型分组展示维度
- **需求来源**: 用户需要在导入序时账后，查看每个科目下挂载了哪些核算维度（银行账户、供应商、客户、项目等）
- **功能逻辑**:
  - 在科目余额表主页面点击"核算维度"按钮或在侧栏选中科目后通过子界面查看
  - 从 `JournalEntry` 的 `dimension` 字段解析出结构化的核算维度
  - 如果科目没有任何核算维度数据（为空），则该界面不显示任何内容
  - 按维度类型分组（如：银行账户、供应商、客户、业态、项目等）
  - 支持点击某一维度值，筛选出该维度值相关的所有序时账分录
- **实现方式**:
  - 可设计为独立的面板/抽屉（Drawer）/对话框
  - 也可作为主区域的一个独立 Tab 页
- **涉及文件**: `frontend/src/components/DimensionPanel.vue`（重构优化）、`backend/routers/journal.py`（维度筛选接口）

## 数据结构

### BalanceSubject（科目余额表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| book_name | String | 所属账套 |
| code | String | 科目编码（如 1002, 1123.99） |
| name | String | 科目名称 |
| year_start_debit | Float | 年初余额（借方） |
| year_start_credit | Float | 年初余额（贷方） |
| period_debit | Float | 本期发生额（借方） |
| period_credit | Float | 本期发生额（贷方） |
| year_total_debit | Float | 本年累计（借方） |
| year_total_credit | Float | 本年累计（贷方） |
| end_debit | Float | 期末余额（借方） |
| end_credit | Float | 期末余额（贷方） |
| dimension | String | 核算维度 |

### JournalEntry（序时账分录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| book_name | String | 所属账套 |
| org | String | 核算组织 |
| period | String | 期间（如 2026年3期） |
| voucher_no | String | 凭证号 |
| summary | String | 摘要 |
| subject_code | String | 科目编码 |
| subject_name | String | 科目名称 |
| debit | Float | 借方金额 |
| credit | Float | 贷方金额 |
| dimension | String | 核算维度 |

## 科目层级规则
- 一级科目：编码不含 `.`，如 `1002`、`1123`
- 二级科目：编码含1个 `.`，如 `1123.99`、`2202.03`
- 三级科目：编码含2个 `.`，如 `2201.01.01`、`2221.12.01`
- 科目树根节点 = 所有一级科目
- 子节点通过编码前缀匹配父节点

## 交互流程
1. 用户启动系统 → 自动加载上次使用的账套
2. 导入科目余额表（Excel）→ 解析存入 MySQL → 侧栏生成科目树
3. 导入序时账（Excel）→ 解析存入 MySQL → 更新统计数据
4. 主页面默认显示完整的科目余额表（表格），支持层级折叠/展开
5. 侧栏点击科目 → 主科目余额表筛选定位到该科目
6. 双击余额表科目名称 → 跳转到序时账视图，显示该科目及其子科目所有分录
7. 点击凭证号 → 弹出凭证详情（借贷格式）
8. 切换到"核算维度"Tab → 显示当前选中科目的核算维度子界面
   - 从序时账中提取并解析该科目所有核算维度
   - 按类型分组展示（银行账户、供应商、客户、项目等）
   - 如果科目无核算维度，界面留白不显示
   - 点击维度值可筛选出对应的序时账分录
9. 可保存当前数据为新账套，下次打开继续使用
10. 预留"导出到底稿"按钮位置，后续对接底稿模板导出

## 风险与应对
| 风险 | 应对方案 |
|------|---------|
| Excel解析格式不匹配 | 改进parser，支持多级表头识别和模糊匹配 |
| 大量数据（2000+科目）加载慢 | 后端分页查询，前端虚拟滚动 |
| 科目层级嵌套过深 | 限制到三级，支持通用 N 级层级 |
| 大数据量序时账查询慢 | 添加数据库索引，按 book_name + subject_code 复合索引 |
